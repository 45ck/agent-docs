## ADR-001 ADR-001: Why this example uses React and TOON-first documentation

Generated from docs/ADR-001.toon

- status: accepted
- path: docs/ADR-001.toon

## Context

The example must be obvious to a broad audience and quick to run while demonstrating planning artifacts.

## Decision

Use React with TypeScript and Vite for familiarity and local state with all documentation in .toon.

## Alternatives

Plain JavaScript was rejected for weaker typing and higher onboarding cost with no clear benefit.

## Consequences

The stack is approachable and demonstrates the workflow but lacks backend persistence.
