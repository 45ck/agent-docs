## SRD-001 System Requirements for the To-Do Example

Generated from docs/SRD-001.toon

- status: accepted
- path: docs/SRD-001.toon

## Architecture Overview

Single-page React app with TypeScript and Vite with state in localStorage for zero backend setup.

## State Model

Each task stores id title notes priority completed dueDate tags and createdAt.

## Data Flow

React component state drives UI and persistence writes every mutation directly to localStorage.

## Filtering Model

Filters include all active completed overdue and high priority plus text search.

## Documentation Flow

All planning and design docs are authored in .toon files under docs and generated outputs are written to generated.

## Tooling

Node 22 React 18 Vite 5 TypeScript 5 plus the local agent-docs CLI package.

## Limitations

No authentication no multi-user sync and no backend API. This keeps setup light and focused on documentation workflow.
